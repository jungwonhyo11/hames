# Hermes for Web Onboarding Notes

## Goal
Make Hermes for Web usable even for people who have never used Hermes CLI or the Telegram bot before.

## Key onboarding promises
1. The UI should feel personal if Hermes memory already exists.
2. The default visual impression should feel warm and memorable.
3. Setup should be explained like a guided tour, not a wall of knobs.
4. Telegram onboarding should be possible from a one-click setup pack.
5. Users should be able to rename the assistant so it feels like their own bot.

## Recommended onboarding surfaces
- Personalized home card
- Setup Packs
- Telegram Onboarding Pack
- README / README.ko
- Private release checklist

## Telegram Onboarding Pack expectations
- explain required credentials or bot setup clearly
- verify whether Telegram is already configured
- provide a first-message workflow for new users
- summarize how WebUI, CLI, and Telegram can share work context
- include a simple first-run checklist for users who have never touched Hermes before
- explain what to do if the user has no Telegram bot token or no gateway yet

## Assistant naming
The UI should let each user rename the assistant.
Examples:
- Gucci
- Luna
- Hermes
- Research Desk

The chosen name should affect:
- top bar title when no session title is active
- composer placeholder
- onboarding tone
